# Contributing to Dr. Kaman Psychology Portal

First off, thank you for considering contributing to this project! It's a personal archive dedicated to making psychological knowledge accessible.

## 📝 Code of Conduct

Please be respectful and constructive in all interactions. We aim to create a welcoming environment for contributors interested in psychology and web development.

## 🚀 How to Contribute

### Reporting Bugs
- **Search existing issues** first to avoid duplicates
- Include:
  - Clear description of the bug
  - Steps to reproduce
  - expected vs actual behavior
  - Screenshots if applicable
  - Django version and Python version

### Suggesting Features
- Open an issue with the tag `enhancement`
- Describe the feature and its benefit to users
- Mention any alternatives you've considered

### Submitting Code
1. **Fork** the repository
2. Create a branch: `git checkout -b feature/your-feature-name`
3. Make your changes and test thoroughly
4. Commit with clear messages: `git commit -m "Add: short description"`
5. Push to your fork: `git push origin feature/your-feature-name`
6. Open a **Pull Request**

### Pull Request Checklist
- [ ] Code follows the project's style
- [ ] Self-review of code completed
- [ ] Comments added for complex logic
- [ ] README updated (if needed)
- [ ] No new warnings or errors
- [ ] Tested migrations work correctly

## 💡 Development Guidelines

### Code Style
- Follow **PEP 8** for Python code
- Use meaningful variable names
- Add docstrings to functions
- Keep Django views focused and modular

### Django Best Practices
- Use Django's built-in authentication where applicable
- Keep models clean and well-documented
- Use class-based views for complex logic
- Follow the MVT (Model-View-Template) pattern strictly

### Testing
- Test on your local server before pushing
- Check that migrations run without errors
- Ensure all media files load correctly
- Test responsive design on mobile devices
